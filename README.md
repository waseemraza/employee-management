# README #


This repository showcases a variety of technical concepts through an evolving hands-on project. It started as a basic Spring Boot + Java setup. The idea is to continuously expand it to demonstrate how different technical topics, tools, patterns, and best practices can be implemented in a real project context.

Each new topic is introduced through a dedicated Pull Request (PR), which is then merged into the main branch. This provides a clear, incremental history of changes and it allows to review the implementation details of each concept step by step through its corresponding PR.

The list below highlights the technical topics added over time:

* Basic `spring boot 4` with `Java 25` project setup.
* `build.gradle` configuration.
* configure liquibase.
* configure integration test using `com.softeq.gradle.itest` gradle plugin.
* use of gradle plugin `java-test-fixtures` to create centralized test data to be used by unit test and integration test.
* configure `docker compose` supported by spring boot internally through spring-boot-docker-compose dependency.
